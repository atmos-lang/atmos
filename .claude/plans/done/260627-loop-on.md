# loop-on

## Goal

Rebuild `loop on` on top of the atmos-native `atm_loop` and drop every
reference to the lua-atmos runtime helpers `loop_on` and `_break_`.

## Background

`loop on PAT IDS { BODY }` currently compiles to a call to the
lua-atmos `loop_on`:

```lua
loop_on(<awt>, function (IDS) BODY end)
```

`loop_on` (lua-atmos `atmos/run.lua:841`) wraps a `while true` await
loop in `catch('atm-loop', ...)` but discards the catch result, so the
`break(v)` value is lost (see plan `260627-loop-on-break.md`).

The atmos-native `atm_loop` (`src/run.lua:78`) already catches
`atm-loop` and re-returns the break value.

## Desugar

```
loop on PAT IDS { BODY }   ==>   loop {
                                     val IDS = await(PAT)
                                     BODY
                                 }
```

When there are no `IDS`, the binding is a bare `await(PAT)` statement.

Resulting codegen (native loop):

```lua
atm_loop(function ()
    for _ in X.iter() do
        local IDS <const> = atm_pin_chk_set(true, false, await(PAT))
        BODY
    end
end)
```

`await` stays: it is the event primitive, not loop machinery.
Only `loop_on` (and the unused `_break_`) leave the codegen.

| `src/prim.lua` | `loop on` branch (~646) | emit a `loop` node with `val IDS = await(PAT)` prepended instead of a `loop_on` call |

## Verification (manual prototype)

`loop { val it = await :X ; ... break(it.n*5) ... }` already yields the
expected `2\n99\n2\n10`, confirming break-value propagation through
`atm_loop`.

## Consequences

- Fixes the `loop on` break-value bug entirely compiler-side.
- Makes the lua-atmos `M.loop_on` fix in `260627-loop-on-break.md`
  unnecessary; that plan can be retired.

## Deferred test

The failing test documenting the break-value bug was removed from
`tst/exec.lua` to keep the suite green (was "loop 13 : break value in
loop on", expecting `2\n99\n2\n10\n`). Re-add it together with the
desugar fix:

```
spawn {
    val x = loop it on :X {
        print(2)
        if it.n == 2 {
            break(it.n * 5)
        }
        print(99)
    }
    print(x)
}
emit :X [n=1]
emit :X [n=2]
```
