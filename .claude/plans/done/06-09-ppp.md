# PPP via `@#` — From-End Indexing

## 1. Decision

Replace the magic ppp keys `[=]` `[+]` `[-]` with plain from-the-end
indexing on `@`, using a bare `#` (= receiver length) as the index key.
No new sigils, no special operator keys.
Peek, set-last, and push all collapse into ordinary index.
Pop is deferred (§6).

## 2. Context

Depends on `06-06-index-table.md` (indexing moved to `@`).

The *ppp operators* (peek, push, pop) today address a vector as a stack
through magic keys inside brackets:

```
vec[=]          ;; peek / set last
vec[+]          ;; push
vec[-]          ;; pop
```

These do not fit the sigil remap (`@` index, `[ ]` table) and read
badly:
`set vec[=] = 30` packs three `=`-like tokens.
They were also the reason a parser `@=` / `@+` / `@-` extension was
proposed — this plan makes that extension unnecessary.

## 3. The rule

FINAL: two bare index markers address a vector relative to its tip
(receiver must be a variable):

```
t@#            ;; ≡ t@(#t)     → last item
t@+            ;; ≡ t@(#t+1)   → next item (append slot)
```

Prefix `#x` (length of `x`) is unchanged.
Arithmetic offsets (`t@(#-1)`, `t@(#+1)`) were DROPPED — only the two
markers above exist; there is no second-to-last form.
Explicit `t@(#t+1)` still works, but as plain indexing (no tip magic).

## 4. Operations

| op           | old (magic key)   | new (tip marker)     |
| ------------ | ----------------- | -------------------- |
| peek / last  | `vec[=]`          | `vec@#`              |
| set last     | `set vec[=] = v`  | `set vec@# = v`      |
| push         | `set vec[+] = x`  | `set vec@+ = x`      |
| pop          | `vec[-]`          | deferred (§6)        |

Example (`exs/exp-13-ppp.atm`):

```
val stk = @{1,2,3}
print(stk@#)           ;; --> 3
set stk@# = 30         ;; @{1, 2, 30}
set stk@+ = 3          ;; @{1, 2, 30, 3}
```

## 5. Removed / cancelled

- Parser: drop the `=` / `+` / `-` special-key branch from `[`-indexing.
- Docs: rewrite `### Peek, Push, Pop` as tip indexing; merged into the
  main `## Indexing` section; migrate `exs/exp-13-ppp.atm`.

Note: an earlier draft cancelled a `@+` extension, but the FINAL design
revives `@+` as the append marker (the `@(#±n)` arithmetic offsets are
the part that was dropped instead).

## 6. Pop — deferred

Pop (read + remove the last element) is the only op that is not plain
addressing.

- Rejected: "`set` returns the previous value" (exchange semantics) — too
  surprising as a general rule.
- Interim spelling, no new feature:

```
val x = t@#
set t@# = nil          ;; removes the last slot (Lua sequence shrinks)
```

- Future: a `pop` helper / method, decided separately.
  Out of scope for this plan.

## Status — IMPLEMENTED, TESTS PASS, COMMITTED

FINAL design = two bare markers `@#` / `@+`; arithmetic offsets dropped.
Committed as `4160117 ! ppp -> t@#, t@+` and `ea84d3e . [] -> @ : manual`.

- [DONE] `src/parser.lua` (`parser_2_suf` `@` branch): pure parse-time sugar,
  desugars in place — NO `tip` node survives:
  - `t@#` → `index{t, uno{#, t}}`         (last item)
  - `t@+` → `index{t, bin{uno{#,t}, +, 1}}` (next/append slot; `#` and `1`
    are synthesized tokens)

    `@(#)` / second-to-last forms. `t@(#)` is now a plain-index parse error

- [DONE] `src/coder.lua` / `src/tosource.lua`: NO special case — the desugared
  `t[#t]` / `t[#t+1]` code naturally. (`f()@#` / `f()@+` are parse errors; bind

## 8. Open questions

- [RESOLVED] Receiver evaluation: the tip receiver is restricted to a
  variable, so inlining `t[#t]` / `t[#t+1]` (which re-reads `t`) is
  side-effect free — no metamethod, no runtime cost. Cost: `f()@#` /
  `f()@+` are rejected; bind first (`val t = f() ; t@#`).
- [RESOLVED] One spelling per op: bare `t@#` / `t@+` only. `t@(#)` is NOT a
  "last" spelling (it's a plain-index error); use `t@#`.
- [RESOLVED] Arithmetic offsets (`#-1`, `#+1` and second-to-last): DROPPED.
- `#` self-length is only meaningful for sequences;
  on a map `#` is 0 / undefined — tip indexing is vector-only.
- Confirm 1-based vectors: `t@#` = `t@(#t)` = last, `t@+` = `t@(#t+1)` =
  append slot.
