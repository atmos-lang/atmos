# 260616-x : migrate removed `atm_*` helpers to lua-atmos `x.*`

## Context

The deep helpers were removed from `src/aux.lua` (git diff):

- `atm_equal` (deep equality)
- `atm_is`    (equivalence)
- `atm_cat`   (concatenation)
- `atm_in`    (membership)

They are assumed to live now in lua-atmos under the `atmos.x` module,
bound in the compiled chunk as the uppercase global `X`
(`src/exec.lua:23`):

- `X.eq`
- `X.is`
- `X.cat`
- `X.xin`

The coder still emits the old global names, so generated programs call
undefined globals at runtime.
Goal: emit `x.*` calls and ensure `x` is in scope.

## Mapping

| op    | coder line | old emit                | new emit            |
| ----- | ---------- | ----------------------- | ------------------- |
| `===` | 82         | `atm_equal(a,b)`        | `X.eq(a,b)`         |
| `=!=` | 84         | `(not atm_equal(a,b))`  | `(not X.eq(a,b))`   |

| `?>`  | 92         | `atm_in(a,b)`           | `X.xin(a,b)`        |
| `!>`  | 94         | `(not atm_in(a,b))`     | `(not X.xin(a,b))`  |

## Risks / open

- Shadowing: resolved by using uppercase `X` (Atmos vars are
  lowercase), so user identifiers no longer collide with the module.
- `_is_` was a lua-atmos global type predicate, now renamed to `X.is`.
  Both emit sites (`src/prim.lua:608`, `src/run.lua:3`) migrated.
  `atm_<match>` accessor is unrelated and kept.
- Keep: `atm_behavior`, `map`, `join`, `concat` remain in `aux.lua`.
