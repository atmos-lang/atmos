# PPP via `@#` — From-End Indexing

## 1. Decision

Replace the magic ppp keys `[=]` `[+]` `[-]` with plain from-the-end
indexing on `@`, using a bare `#` (= receiver length) as the index key.
No new sigils, no special operator keys.
Peek, set-last, and push all collapse into ordinary index.
Pop is deferred (§6).

## 2. Context

Depends on `06-06-index-table.md` (indexing moved to `@`).

The *ppp operators* (peek, push, pop) today address a vector as a stack
through magic keys inside brackets:

```
vec[=]          ;; peek / set last
vec[+]          ;; push
vec[-]          ;; pop
```

These do not fit the sigil remap (`@` index, `[ ]` table) and read
badly:
`set vec[=] = 30` packs three `=`-like tokens.
They were also the reason a parser `@=` / `@+` / `@-` extension was
proposed — this plan makes that extension unnecessary.

## 3. The rule

A bare `#` (no operand) inside an `@`-index means "length of the
receiver":

```
t@#            ;; ≡ t@(#t)   → last element
t@(#-1)        ;; second-to-last
t@(#+1)        ;; one past the end (push slot)
```

Prefix `#x` (length of `x`) is unchanged.
The new case is `#` with no operand, valid only as an index key.
Bare `t@#` is the terse form for the common "last"; parentheses are used
when combined with arithmetic (`t@(#-1)`, `t@(#+1)`).

## 4. Operations

| op           | old (magic key)   | new (from-end index) |
| ------------ | ----------------- | -------------------- |
| peek / last  | `vec[=]`          | `vec@#`              |
| set last     | `set vec[=] = v`  | `set vec@# = v`      |
| second-last  | —                 | `vec@(#-1)` (free)   |
| push         | `set vec[+] = x`  | `set vec@(#+1) = x`  |
| pop          | `vec[-]`          | deferred (§6)        |

Example (was `exs/exp-13-ppp.atm`):

```
val stk = @{1,2,3}
print(stk@#)           ;; --> 3
set stk@# = 30         ;; @{1, 2, 30}
set stk@(#+1) = 3      ;; @{1, 2, 30, 3}
```

## 5. Removed / cancelled

- Parser: drop the `=` / `+` / `-` special-key branch from `[`-indexing.
- The deferred parser `@=` / `@+` / `@-` extension is CANCELLED.
- Docs: rewrite `### Peek, Push, Pop` as from-end indexing;
  migrate `exs/exp-13-ppp.atm`.

## 6. Pop — deferred

Pop (read + remove the last element) is the only op that is not plain
addressing.

- Rejected: "`set` returns the previous value" (exchange semantics) — too
  surprising as a general rule.
- Interim spelling, no new feature:

```
val x = t@#
set t@# = nil          ;; removes the last slot (Lua sequence shrinks)
```

- Future: a `pop` helper / method, decided separately.
  Out of scope for this plan.

## 7. Change points

| File / place        | Change                                              |
| ------------------- | --------------------------------------------------- |
| `lexer.lua`         | none (`#` already lexes)                            |
| `parser.lua` 2_suf  | accept bare `#` as an index key (after `@`, and     |
|                     | inside `@(…)`) → a self-length node; remove the     |
|                     | `=`/`+`/`-` key branch                              |
| `coder.lua`         | emit bare `#` as `#<receiver>`; `t@(#-1)` →          |
|                     | `t[#t - 1]`; bind the receiver once (temp) to avoid |
|                     | re-evaluating a side-effecting `t`; lvalue path too |
| `tosource.lua`      | print the index back (`#`, `#-1`, `#+1`)            |

## Status — IMPLEMENTED

- [DONE] `src/parser.lua`: `@#` is pure parse-time sugar — DESUGARS in place
  to a plain index by `#t` (`index{t, uno{#, t}}`); `@(#±n)` →
  `index{t, bin{uno{#,t}, ±, n}}`. No `tip` node survives. Receiver must be a
  variable (`acc`) — else `index error : tip requires a variable`.
  - Tip vs `#x`: the `@` branch in `parser_2_suf` consumes `#`, then PEEKS the
    next token — `)`/`+`/`-` => operandless => tip (`#` of `e`); else `#x`
    (normal length via `parser_4_pre`). Then one optional `±` offset, built by
    hand. ALL LOCAL — no global flag, `parser_4_pre` is untouched. (Earlier
    drafts used a `TIP_E` global + a metamethod; both removed.)
  - So `t@(#+1)` (tip) and `t@(#t+1)` (explicit `#t`) both parse; `t@(#x)` =
    index by length of x. Limit: a single `±` offset only.
  - VERIFY: this last refactor needs a full suite run to confirm green.
  - The `[` `=`/`+`/`-` ppp key branch is removed (whole `[` index gone now).
- [DONE] `src/coder.lua` / `src/tosource.lua`: NO special case — both are
  back to the plain `index` one-liner; the desugared `t[#t ± n]` codes
  naturally. (`f()@#` is a parse error; bind to a var first. tosource prints
  the desugared `t@((#t))`, which is fine — no test asserts the sugar.)
- [DONE] `src/run.lua`: no metamethod (`meta_table = {}`); ppp machinery gone.
- [DONE] migrated call sites: `tst/exec.lua`, `doc/exs/exp-13-ppp.atm`,
  `doc/manual.md` (`### Peek, Push, Pop` -> `### Tip Indexing`, nav).

  (left to the doc build).
- Deferred: a first-class `pop` (still the two-step interim).

## 8. Open questions

- [RESOLVED] Receiver evaluation: the tip receiver is restricted to a
  variable, so inlining `t[#t ± n]` (which re-reads `t`) is side-effect free
  — no metamethod, no runtime cost. Cost: `f()@#` is rejected; bind first
  (`val t = f() ; t@#`).
- Accept both `t@#` and `t@(#)` spellings for "last"?
  (bare for the hot path, parens when arithmetic follows).
- `#` self-length is only meaningful for sequences;
  on a map `#` is 0 / undefined — from-end indexing is vector-only.
- Confirm 1-based vectors: `t@#` = `t@(#t)` = last, `t@(#+1)` = push slot.
