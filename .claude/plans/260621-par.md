# Plan: `par_and`/`par_or` → `par :all`/`par :any`

> Status: IMPLEMENTED — §1-§5 done. Pending: user runs the test suite;
> regenerate `manual-out.md` from `manual.md`.


## Context

Atmos has three parallel combinators: bare `par` (never rejoins),
`par_and` (rejoins after all), `par_or` (rejoins after any).
`par_and`/`par_or` are separate keywords.
We want to eliminate those two keywords and express the variant as a
tag on `par`:

    par_and { ... }   ->   par :all { ... }
    par_or  { ... }   ->   par :any { ... }

Bare `par` stays as-is.
Tags `:all`/`:any` already lex independently and are already used the
same way in `await :all`/`await :any` (`src/await.lua:39`), so this
reuses an existing convention.
The runtime functions stay named `par_and`/`par_or` (they live in the
external `lua-atmos` package at `/x/lua-atmos`, out of scope); the
compiler keeps emitting calls to those names.

## Approach

### 1. Compiler core (`src/`)

| file | place | description |
| ------------ | -------------------- | ------------------------------------------ |
| global.lua   | `KEYS` (line 34)     | remove `'par_and'`, `'par_or'`             |
| prim.lua     | par branch (669-689) | accept `par` + optional `:all`/`:any` tag  |

`prim.lua` new logic (replacing the `check`/`accept` of the 3 keywords):

```lua
-- pars, watching
elseif check('par') or check('watching') then
    -- par :all -> par_and (rejoin all); :any -> par_or (rejoin any)
    if accept('par') then
        local par = 'par'
        if accept(':all', 'tag') then
            par = 'par_and'
        elseif accept(':any', 'tag') then
            par = 'par_or'
        end
        local fs = { parser_block() }
        ...unchanged (with-blocks, proto wrap, call node str=par)...
```

`accept(':all', 'tag')` / `accept(':any', 'tag')` mirror `await.lua:39`.
Coder (`coder.lua:111`) and `tosource.lua:43` need no change — they emit
the `call` node whose fn name is `par`/`par_and`/`par_or`.
