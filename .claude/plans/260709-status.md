# status -- query a task's lifecycle state

Tracking: https://github.com/atmos-lang/atmos/issues/32 (empty body)

## Goal

Give atmos-lang a synchronous, non-blocking way to read a task's
current lifecycle state (alive / awaiting / toggled-off / terminated),
filling the gap where `toggle` can set state but nothing reads it back.

## Status: DRAFT -- blocker resolved (runtime surveyed); design open

## Runtime survey (RESOLVED)

Runtime source read at `/x/lua-atmos/atmos/atmos/` (checked out).

A runtime `status` ALREADY EXISTS but is partial and unreachable from
atmos-lang:

    -- run.lua:105
    function M.status (t)
        return coroutine.status(t._.th)   -- suspended|dead|running|normal
    end

- Exported as `atmos.status` (init.lua:11) but NOT a global, and the
  atmos-lang compiler does not know it -> unreachable from `.atm` today.
- Returns raw Lua coroutine vocabulary and IGNORES the atmos overlay
  `t._.status` (`nil` | `'aborted'` | `'toggled'`, init.lua:423).

A task therefore has TWO independent state axes:

| axis      | field                       | values                        |
|-----------|-----------------------------|-------------------------------|
| lifecycle | `coroutine.status(t._.th)`  | suspended / dead / running / normal |
| overlay   | `t._.status`                | nil / aborted / toggled       |

Implication: the real work is a lua-atmos (upstream) change to merge the
two axes; the atmos-lang side is thin.
NOTE: lua-atmos is a SEPARATE repo -- I will not edit it; upstream items
are documented here as a dependency.

## What exists now (no gap-filler today)

| form                 | gives                                 |
|----------------------|---------------------------------------|
| `t ?? :xtask`        | is a running task instance (type)     |
| `T ?? :task`         | is a task prototype (type)            |
| `ts ?? :tasks`       | is a task pool (type)                 |
| `await(t)`           | BLOCKS until terminated, yields value |
| `await(:any/:all ts)`| pool termination                      |
| `toggle t(on/off)`   | SET reactivity; cannot be read back   |

`?? :xtask` is a type check, not a liveness check: it stays true after
the task terminates.

## Design (open -- confirm before coding)

### Surface syntax

Proposed: a primitive call `status(t)` returning a tag.

    val s = status(t)
    if s == :done { ... }

Alternative to weigh: extend `??` (e.g. `t ?? :done`) to stay in the
existing query family instead of adding a call primitive.
Recommendation: prefer `status(t)` -- `??` is type identity, lifecycle
is a different axis and overloading it muddies both.

### Value domain -- mapping the two axes

The runtime CAN distinguish all cases; overlay wins over lifecycle.
Proposed merge (return atmos tags, not raw Lua strings):

| runtime condition            | `status(t)` |
|------------------------------|-------------|
| `t._.status == 'aborted'`    | `:aborted`  |
| `t._.status == 'toggled'`    | `:toggled`  |
| coroutine `'dead'`           | `:done`     |
| coroutine `'suspended'`      | `:awaiting` |
| coroutine `'running'`        | `:running`  |
| coroutine `'normal'`         | `:running`  (ancestor on the call stack) |

Open choice: return atmos tags (idiomatic, e.g. `if s==:done`) vs the
raw Lua string from `M.status` (cheaper, leaks Lua vocabulary + drops
overlay).
Recommendation: tags, merged upstream in `run.status`.

### Philosophy caveat

Atmos favors `await`/`emit` reaction over polling.
`status(t)` invites poll-loops the language usually designs away.
Document the intended use (one-shot inspection, debugging, guards) and
steer readers toward `await` for reactive flows.

## Changes

### Path A -- minimal (bare global call, no new syntax)

`status(t)` is a plain 1-arg call; atmos identifiers already compile to
Lua globals (like `tasks(2)`).
If the runtime exposes a global `status`, `.atm` code calls it with ZERO
compiler change.

| repo      | file            | change                                   |
|-----------|-----------------|------------------------------------------|
| lua-atmos | `atmos/init.lua`| add global `status = run.status` (upstream) |
| lua-atmos | `atmos/run.lua` | enrich `M.status` to merge overlay + return tags (upstream) |
| this repo | `doc/manual.md` | Tasks chapter: document `status`, value domain, caveat |

### Path B -- first-class primitive (only if we want checks/tags here)

Adds parse-time validation + tag mapping in the compiler instead of
upstream.

| file            | place                | change                              |
|-----------------|----------------------|-------------------------------------|
| `src/lexer.lua` | natives / keywords   | recognize `status`                  |
| `src/prim.lua`  | primitive parse (~137)| parse `status(Expr)` like siblings |
| `src/coder.lua` | primitive codegen    | emit runtime call + tag mapping     |
| `src/run.lua`   | shim                 | `atm_status` merging both axes      |
| `doc/manual.md` | Tasks chapter        | document `status`                   |

Grammar sketch (Path B):

    Status : `status´ `(´ Expr `)´

Recommendation: **Path A** -- the runtime already owns this state; keep
the merge upstream and the surface a plain call.
Fall to Path B only if we want argument checks (non-task / proto / pool)
enforced by the compiler.
